Overview

Aran API Sentinel is a comprehensive API security and governance platform designed to protect, monitor, and manage your API ecosystem. Built with modern technologies including Next.js 15, Bun runtime, and PocketBase, it provides enterprise-grade security features with a developer-friendly interface.

🚀 Key Highlights:
  • Multi-tenant architecture with company isolation
  • Hybrid AI-powered threat detection
  • Real-time API monitoring and analytics
  • MCP (Model Context Protocol) security
  • Modern, responsive UI built with React

Architecture

Installation

Prerequisites

  • Bun runtime (v1.0.0+)
  • Git
  • Modern web browser

Quick Start

# Clone the repository
git clone https://github.com/radhi1991/aran.git
cd aran

# Install dependencies
bun install

# Start the development server
bun run dev:full

# The application will be available at:
# Frontend: http://localhost:9002
# Backend: http://localhost:8090

Environment Setup

Create a .env.local file in the root directory:

# AI Provider API Keys (Optional)
OPENAI_API_KEY=your_openai_key
GEMINI_API_KEY=your_gemini_key
ANTHROPIC_API_KEY=your_anthropic_key

# Database Configuration
DATABASE_URL=file:./data.db

# Application Settings
NEXT_PUBLIC_APP_URL=http://localhost:9002

Usage Guide

Dashboard Overview

The main dashboard provides a comprehensive view of your API security status, including:

  • Total APIs discovered and monitored
  • Active security scans and threats detected
  • Real-time activity feed
  • Quick action buttons for common tasks

API Discovery

Use the API Discovery feature to:

  • Scan your network for new API endpoints
  • Analyze security vulnerabilities
  • Monitor API performance and usage
  • Generate security reports

Threat Detection

The AI-powered threat detection system:

  • Monitors API traffic for anomalies
  • Uses multiple AI providers for analysis
  • Provides detailed threat reports
  • Offers remediation recommendations

AI Providers Management

Manage your AI analysis providers:

  • Enable/disable specific providers
  • Monitor performance and success rates
  • View analysis logs and history
  • Configure provider priorities

API Reference

Authentication

All API endpoints require authentication. Use the PocketBase authentication system:

POST /api/collections/users/auth-with-password
{
  "identity": "user@example.com",
  "password": "password"
}

API Endpoints

# API Discovery
GET /api/collections/api_endpoints/records

# Threat Detection
POST /api/collections/threats/records

# AI Analysis
POST /api/ai/analyze

# User Management
GET /api/collections/users/records

Deployment

GitHub Pages Deployment

This application is automatically deployed to GitHub Pages using GitHub Actions. The deployment process:

  1. Builds the application using Bun
  2. Runs tests to ensure quality
  3. Generates static documentation
  4. Deploys to GitHub Pages
🌐 Live Demo: The application is deployed at https://radhi1991.github.io/aran/

Local Development

# Development mode
bun run dev:full

# Production build
bun run build

# Start production server
bun run start

# Run tests
bun test

# Database operations
bun run db:seed
bun run db:migrate
